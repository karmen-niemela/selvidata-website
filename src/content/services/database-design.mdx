## What I deliver

- **Schema design** — normalized, well-documented data models that grow with your product
- **Query optimization** — identify and fix slow queries dragging down your application
- **Indexing strategy** — the right indexes for your access patterns without bloating writes
- **Migration planning** — safe, zero-downtime schema migrations for production databases
- **Architecture review** — evaluate your current setup and recommend improvements

## Project examples

### B2B SaaS — scaling past growing pains

A pattern that shows up consistently at the growth stage: the database schema was designed for an early product and a small customer base, and it's now causing problems. API timeouts are creeping up, reports are slow to load, and engineering is spending more time on performance fires than new features.

This type of engagement starts with a database audit — identifying missing indexes, N+1 query patterns, and schema design decisions that made sense at launch but don't hold up under load. The output is a prioritized set of fixes, from quick index additions to query rewrites, with measurable improvements to response times and infrastructure costs.

### B2B SaaS — multi-tenant schema design

Early-stage SaaS products often start with a schema that doesn't properly account for multi-tenancy — data isolation is informal, there's no clean tenant boundary, and adding enterprise features like role-based permissions or audit logging requires reworking the core model.

This type of project involves redesigning the schema with proper tenant isolation, row-level security, and a structure that supports compliance requirements without sacrificing query performance. The goal is a data model that can support enterprise sales without requiring a painful re-architecture later.
